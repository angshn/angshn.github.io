# 红黑树的基本性质

红黑树有五个基本性质：

  1. 每个节点或是红色的，或是黑色的。
  2. 根节点是黑色的
  3. 每个叶节点都是黑色的。（Nil）节点
  4. 如果一个节点是红色的，那么他的节点都是黑色的。
  5. 每个节点到后代所有叶节点的简单路上经过的黑色节点数目相同。

# 插入

红黑树的插入和基本的BST一样，都是先找到给定值在树中的位置，然后新建节点node，新节点的颜色统一为红色，因此红黑树的节点插入稍微有点复杂，因为红色节点的插入可能会破坏红黑树的性质，例如性质[2,4]。因此需要对插入后的结果进行修正。

红黑树的修正一般都设计到父节点，兄弟节点，叔节点等。为了方便首先定义一些记号：

  1. pa(x)：表示x的父亲
  2. uc(x): 表示x的叔节点
  3. sib(x): 表示x的兄弟节点
  4. x: 表示新插入的节点
  5. red(x): 表示x节点的颜色是不是红色

# 插入情况枚举

红色节点的插入需要根据周围节点的颜色来进行平衡调整，红色节点插入之后有3中情况：

  1. red(pa(x))==true，父节点是红色。爷节点一定是黑色。此时破坏了红黑树的性质4，需要进行调整。

    a. red(uc(x))==true，叔节点是红色。此时需要将父叔节点的颜色与爷节点颜色对调。处理结束后将节点指向爷节点，继续处理。

    b. red(uc(x))==false,叔节点是黑色。当前节点x和父节点的顺序和父节点和爷节点的循序一致，公示表示为

    转为a步骤急需处理。

    c. 如过不满足，首先要满足该条件，在执行b步骤。

    如果pa(x)==left(pa(pa(x)))时，也就是父节点是爷节点的左节点时，x是父节点的右节点是（有对称情况），使用左旋将方向同轴，也就是都在左节点或者都在右节点（对称情况）。此时可转为b步骤继续执行

  2. red(pa(x))==true，父节点是黑色节点，直接插入，不会影响任何东西。

# 删除节点

在红黑树中删除一个值时可以先像BST那样基于比较遍历获取目标节点p，此时需要根据目标节点p周围节点的颜色来处理。以下分情况讨论

# 1. 删除红色底部节点

删除一个红色节点不会对红黑树有任何影响，直接使用他的子树之一来替代p即可。

# 2. 非根部红色节点且有单个子节点

如果要删除的节点是红色且有一个子节点，此时使用子节点直接替换即可。

# 3. 非根红色有双节点

在以p为根的子树中找一个p的值的后继q，用q的值替换p的链接和值，q通常在右子树的最左边节点。此时需要处理被替换的节点q。转为其他1-4的处理方式继续处理。

# 4. 删除黑色节点

删除黑色节点时要根据兄弟节点来进一步划分。同一目标是将兄弟节点变为黑色，如果是红色就变成黑的再处理。

1. p的兄弟是红色

red(sib(x))==true，此时兄弟节点w必然有黑色子节点，如果没有则违反红黑树的第四条性质。

处理步骤如下：

  1. 设置父红，兄黑，此时破坏了红黑树的性质。
  2. 对兄弟节点执行一次左旋
  3. w指向新的兄弟节点

2. p的兄弟w是黑色，其子节点全为黑色

处理步骤如下：

  1. 将兄弟节点染红色
  2. 将x指向父亲pa(x)，即x=pa(x)
  3. 继续处理父节点

3. p的兄弟w是黑色，但是有一个红色子节点

处理步骤如下：

  1. 红子节点和w颜色互换
  2. 右旋将黑色节点上提，生成黑->红->黑的单链
  3. sib指向新的节点
  4. 此时兄弟节点的右子节点变红，w依然是黑的。

4. w是黑，但是w的右子节点时红色

这一步可以从情况3转移过来或者本来就是这种情况。

处理步骤如下：

  1. w设置为父亲的颜色
  2. 父节点染为黑色
  3. w的右子节点染为黑色
  4. 左旋
  5. x设置为树根，此时就终止了

# 删除

private void deleteEntry(Entry<K,V> p) {
    //
    size--;

    // If strictly internal, copy successor's element to p and then make p
    // point to successor.
    // 如果是一个内部节点且双子节点都在，那么直接取当前节点的后继作为当前节点的值
    if (p.left != null && p.right != null) {
        Entry<K,V> s = successor(p);
        p.key = s.key;
        p.value = s.value;
        //此时后继结点就是下一个要删除的点
        p = s;
    } // p has 2 children

    //优先用左节点做替换，没有左子结点用右节点
    //此时只有一种情况，p指向的节点最多只有一个子节点
    // Start fixup at replacement node, if it exists.
    Entry<K,V> replacement = (p.left != null ? p.left : p.right);
    //如果p有子节点
    if (replacement != null) {
        // Link replacement to parent
        //开始执行替换，用replacemnt来代替p
        //父亲替代
        replacement.parent = p.parent;
        if (p.parent == null)
            root = replacement;
        //孩子替换
        else if (p == p.parent.left)
            p.parent.left  = replacement;
        else
            p.parent.right = replacement;
        //清空要删除节点的指针
        // Null out links so they are OK to use by fixAfterDeletion.
        p.left = p.right = p.parent = null;
        // 如果当前节点是黑色节点的话就要处理了，删除一个黑色节点相当麻烦
        // Fix replacement
        if (p.color == BLACK)
            // 从replacement开始自底向上来调整红黑树的平衡性
            fixAfterDeletion(replacement);
    } else if (p.parent == null) { // return if we are the only node.
        root = null;
    } else { //  No children. Use self as phantom replacement and unlink.
        //p节点是黑色，但是没有子节点，此时一样要进行调整。
        if (p.color == BLACK)
            fixAfterDeletion(p);
        //如果不是黑色节点的话就说明是红色的，此时直接删除不会对树造成影响。
        if (p.parent != null) {
            if (p == p.parent.left)
                p.parent.left = null;
            else if (p == p.parent.right)
                p.parent.right = null;
            p.parent = null;
        }
    }
}

稍微总结一下，红黑树的删除需要区分删除红色节点还是黑色节点，如果是红色就直接删除，不会对树有影响，如果是黑色节点的话就要进行适当的调整了。

下面来看删除时的红黑树性质维护，以TreeMap的fixAfterDeletion的左边为例。

private void fixAfterDeletion(Entry<K,V> x) {
    while (x != root && colorOf(x) == BLACK) {
        if (x == leftOf(parentOf(x))) {
            // 获取兄弟节点w，源码里是sib，更清晰
            Entry<K,V> sib = rightOf(parentOf(x));
            //兄弟是红色
            // 按照情况1来应对，直接设置兄黑父红左旋
            // 左旋后新的兄弟节点就是黑色了
            if (colorOf(sib) == RED) {
                setColor(sib, BLACK);
                setColor(parentOf(x), RED);
                rotateLeft(parentOf(x));
                sib = rightOf(parentOf(x));
            }
            // 此时兄弟是黑色，他的孩子也都是黑色，这时候直接设置兄弟是红的，然后处理节点上升
            if (colorOf(leftOf(sib))  == BLACK &&
                colorOf(rightOf(sib)) == BLACK) {
                setColor(sib, RED);
                x = parentOf(x);
            } else {
                // 兄弟是黑色的，但是他的孩子里面有红的
                // 左节点时红的，将左红孩子调整到右边
                // 左孩子和父节点颜色互换，右旋将左红孩子上升，父下降
                if (colorOf(rightOf(sib)) == BLACK) {
                    setColor(leftOf(sib), BLACK);
                    setColor(sib, RED);
                    rotateRight(sib);
                    sib = rightOf(parentOf(x));
                }
                //这是右孩子是红的，左孩子是黑色的
                // 兄弟和父亲交换颜色
                // 兄弟右孩子设置为黑
                // 左旋将兄弟拉上来称为新的父亲
                // 到这种情况就直接结束了
                setColor(sib, colorOf(parentOf(x)));
                setColor(parentOf(x), BLACK);
                setColor(rightOf(sib), BLACK);
                rotateLeft(parentOf(x));
                x = root;
            }
        } else { // 右子节点的情况
    }

    setColor(x, BLACK);
}
