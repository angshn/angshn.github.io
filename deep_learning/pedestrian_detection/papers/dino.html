DINO DETR with Improved DeNoising Anchor Boxes for End-to-End Object Detection

Abstract

Introduction

DINO是DN-DETR的进阶版本。

Related Works

Methods

3.1 前置内容

  1. Conditinal DETR和DAB DETR都将query分为positional part和content part,这两个部分也被记为Positional Query和Content Query。在DAB-DETR中将Positional Query用4D 锚框anchor box (x,y,w,h)来表示。明确的anchor box定义使decoder可以更容易地逐层优化（refine）;
  2. DN-DETR认为[[Bipartite Matching|二分匹配]]过程的不稳定导致了DETR-like无法快速收敛。为了解决这个问题，DN-DETR把noised GT labels和noised GT boxes送入detr的decoder，以此来提供额外的loss，称为DN-loss
  3. Deformable DETR中有两个重要的技术——参考点和迭代性边框优化。
    1. 参考点使每个query 只关注局部的一些key和value，而不用关注全局。使得detr很早收敛。
    2. 迭代性边框优化采用相邻decoder layer梯度分离方法，也就是每一个layer只累计相邻层的梯度。本文称其为 look forward once。

^1a2c6f

和DAB-DETR和DN-DETR一样，DINO使用positional queries作为动态锚框 Dynamic Anchors（这里的query就是4D anchor）。此外还融合了Deformable DETR的deformable attention和look forward once机制来进一步改进性能。 此外，还采用了Deformable的[[Query Selection#^f62dea | Query Selection]]来更好的初始化位置查询positional queries。

3.3 CDN-Contrastive DeNoising Training

每一个GT box都会生成一个positive和一个negetive query，这样一个CDN中就有有个queries。和DN-DETR类似，DINO也类似的采用了多个CDN group改进方法的性能和表现。我们是期望negetive query能够检测到no object的信息。

从[[#^949f9c|Fig.1. 结构图]]可以看出CDN适合query一起送入decoder训练的。

分析

DINO能够工作的原因是它能够抑制混淆confusion，同时选择高质量的query或者anchor预测bounding box。confusion通常出现在多个anchor都和同一个目标接近的时候，在这种情况下，模型很难抉择到底选择哪一个anchor来预测。 Confusion可能会导致两个问题：一个就是重复预测，虽然DETR-like的检测器可以通过self-attention机制和Set-based loss抑制重复预测，但是这种能力是有限的。在使用了CDN模块取代DN模块之后，模型能够区分anchors之间的轻微差别并且避免重复预测。另一个问题是距离GT box更远的anchor被选中，CDN模块让模型学会了如何拒绝距离更远的anchor。

为了评估CDN模块的有效性，作者设计了Average Top-K Distance metric来评估。对于每一个GT和它对应的anchor box，使用距离衡量他们之间的差距，然后选取前k个最大的值取平均作为ATD的值。这个值越小越好。

3.4 Mixed Query Selection

在vanilla detr和DN-DETR中，decoder queries都是静态的embeddings，而没有融合或提取任何的encoder提取的单个图像的特征，也就是说decoder query和图片没有关系。

^13ab5d

动态4D anchor格式的query和decoder的positional query紧密相关（？），这个anchor可以被query selection策略改进。作者基于static query和query selection方法提出了混合查询选择方法——mixed query selection approach，正如[[#^13ab5d|Fig.5. c]]所示。作者仅仅使用top-K features初始化anchor box，但是同时保留了content queries——也就是Query Embedding，是一个(Size,256)的Tensor。

从被选择的top-K个encoder features生成的dynamic queries是一个初步结果，可能会使decoder产生误解或者被误导。例如，一个feature可能会包含多个目标或者是目标的一部分。在Mixed Query Selection方法中使用top-K selected features增强positional queries（4D anchor），同时也像其他DETR方法那样保留了Content queries。

3.5 Look Forward Twice

Deformable DETR的decoder layer i的参数仅被当前layer的 生成的Loss所更新，表示当前decoder层的预测。这里作者猜想能否用后面的层的box 信息可能会对前一层的box生成有所帮助。因此坐着提出了 Look Forward Twice参数，这里的Twice是相对于Deformable DETR的仅使用当前layer的更新参数相对而言的，也就是说多向前看了一些内容。 每层生成的会被使用两次，一次是用在，另一次是用在上。

为什么要用当前层生成的用在两个地方？

最终预测 被两个因素所决定：初始框和框偏差，Look Forward Once只是优化了后者。layer i和layer i-1之间的梯度信息是分离的。

总的来说就是利用后面的layer i+1和当前层layer i的Loss更新当前层的参数来改善当前的box生成效果。
